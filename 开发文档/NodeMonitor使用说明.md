# Node Monitor 使用说明

## 功能概述

Node Monitor 是一个类似于 Android Monitor 的界面节点识别器，可以帮助开发者实时查看和分析当前页面的所有UI节点信息。

## 主要功能

### 1. 系统顶层浮动按钮
- **位置**: 屏幕右上角
- **功能**: 
  - 主按钮：开启/关闭监控模式
  - 节点计数：显示当前页面节点总数
  - 列表按钮：打开侧滑面板查看详细信息

### 2. 实时节点监控
- **自动刷新**: 每秒自动更新当前页面的节点信息
- **层级显示**: 显示节点的层级深度关系
- **属性标识**: 标识节点是否可点击、可滚动等属性

### 3. 侧滑面板
- **节点列表**: 显示所有检测到的UI节点
- **快速筛选**: 按类型、属性等筛选节点
- **详细信息**: 点击节点查看完整属性信息

### 4. 节点详情页
- **基本信息**: 类名、文本、描述、资源ID
- **位置信息**: 屏幕坐标、宽高尺寸
- **交互属性**: 可点击、可滚动、启用状态等

## 使用步骤

### 1. 启用无障碍服务
1. 打开系统设置 → 无障碍
2. 找到 "AutoFlow" 服务
3. 开启无障碍权限

### 2. 启动监控器
1. 在 AutoFlow 主界面点击左上角的 👁️ 按钮
2. 或者通过菜单启动 Node Monitor

### 3. 查看节点信息
1. 启动监控后，右上角会出现浮动按钮
2. 点击列表按钮打开侧滑面板
3. 在面板中浏览所有节点信息
4. 点击具体节点查看详细属性

## 技术实现

### 核心组件
- `NodeMonitorService`: 主服务，管理浮动窗口和节点监控
- `MonitorUI.kt`: UI组件，包含浮动按钮和侧滑面板
- `NodeMonitorManager`: 管理器，控制监控状态

### 权限要求
- `SYSTEM_ALERT_WINDOW`: 系统浮动窗口权限
- `BIND_ACCESSIBILITY_SERVICE`: 无障碍服务权限

### 数据结构
```kotlin
data class NodeInfo(
    val className: String,        // 节点类名
    val text: String,            // 显示文本
    val contentDescription: String, // 内容描述
    val resourceId: String,      // 资源ID
    val bounds: Rect,           // 屏幕位置
    val isClickable: Boolean,   // 是否可点击
    val isScrollable: Boolean,  // 是否可滚动
    val isEnabled: Boolean,     // 是否启用
    val depth: Int             // 层级深度
)
```

## 使用场景

1. **UI自动化测试**: 快速定位页面元素
2. **无障碍开发**: 检查无障碍属性设置
3. **界面调试**: 分析界面层级结构
4. **性能优化**: 识别复杂的布局层级

## 注意事项

1. **性能影响**: 监控模式会消耗一定系统资源，建议使用完毕后及时关闭
2. **权限管理**: 首次使用需要手动授予浮动窗口权限
3. **兼容性**: 支持 Android 5.0 (API 21) 及以上版本
4. **隐私保护**: 仅在本地分析节点信息，不会上传任何数据

## 快捷操作

- **快速开关**: 点击主浮动按钮
- **快速查看**: 点击节点计数查看总数
- **快速定位**: 在侧滑面板中点击节点名称
- **快速关闭**: 点击面板外区域或返回按钮
