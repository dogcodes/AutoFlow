# 项目概述

## 🎯 项目目标
开发一个类似Dify的Android可视化工作流系统，支持拖拽编辑、自动化执行、配置导入导出等功能。

## 🏗️ 技术架构

### 核心技术栈
- **UI框架**: Jetpack Compose
- **架构模式**: MVVM + StateFlow
- **依赖注入**: 原生Android (可选Hilt)
- **网络请求**: OkHttp3
- **数据序列化**: Gson
- **自动化**: AccessibilityService

### 项目结构
```
com.carlos.autoflow/
├── MainActivity.kt                 # 应用入口
├── ui/
│   ├── theme/                     # 蓝色主题系统
│   └── components/                # 通用UI组件
├── workflow/
│   ├── models/                    # 数据模型
│   ├── viewmodel/                 # 状态管理
│   └── ui/                        # 工作流UI
├── accessibility/                 # 无障碍服务
├── recorder/                      # 操作录制引擎
└── billing/                       # 商业化功能
```

## 🎨 设计理念

### UI风格
- **主色调**: 蓝色系 (#1976D2)
- **设计风格**: 中国大陆流行的现代简约风
- **交互方式**: 拖拽 + 点击的直观操作
- **视觉效果**: 圆角卡片 + 渐变背景

### 用户体验
- **学习成本低**: 类似流程图的直观界面
- **操作流畅**: 60fps的动画和交互
- **功能完整**: 从创建到执行的完整闭环
- **安全可靠**: 完善的权限管理和错误处理

## 📊 风险评估

### 技术风险
- **无障碍服务限制**: Android系统对无障碍服务的限制可能影响功能
- **应用兼容性**: 不同Android版本和厂商的兼容性问题
- **性能瓶颈**: 大型工作流的渲染和执行性能

### 解决方案
- 提供降级方案和替代功能
- 充分测试主流设备和系统版本
- 实现懒加载和性能优化机制
