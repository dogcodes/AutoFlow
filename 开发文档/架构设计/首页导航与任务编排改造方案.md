# 首页导航与任务编排改造方案

## 1. 目标

本次改造目标不是删减功能，而是重组首页信息架构，让 AutoFlow 从“单页编辑器工具”调整为“任务管理 + 工作流编排”的产品结构。

改造后优先满足以下目标：

- 首页先看到任务列表，而不是直接进入画布编辑器
- 保留现有工作流编辑能力，作为独立一级入口继续使用
- 录制能力先保留，但不作为主流程依赖
- 低频工具和配置统一收纳，避免主页面持续膨胀
- 为后续任务定时、多个工作流管理预留扩展空间

## 1.1 当前实现状态

截至当前版本，以下内容已经落地：

- `MainActivity` 启动入口已切换到首页容器
- 首页已改为底部 4 个 Tab：`任务 / 编排 / 录制 / 更多`
- `任务` 已作为默认首页
- `编排` 已复用现有 `WorkflowEditor`
- `录制` 已接入现有 `RecordingControlPanel`
- `更多` 已收纳历史记录、节点监控器、示例应用、设置、关于、许可证管理
- 任务列表已支持基础操作：执行、停止、编辑、删除
- 任务数据已通过本地仓库持久化
- 执行历史已改为共享实时数据源，任务页与历史页可同步观察更新

以下内容尚未完整落地：

- `定时` 当前只有入口，尚未完成正式配置页面和任务级持久化
- 多工作流并行执行尚不支持
- `WorkflowEditor` 内部旧的导航壳仍保留，属于过渡代码，尚未清理

## 2. 导航结构

首页采用底部 4 个 Tab：

- 任务
- 编排
- 录制
- 更多

默认首页为 `任务`。

### 2.1 Tab 职责

#### 任务

任务页是产品主入口，负责任务管理和快捷操作。

主要内容：

- 展示已保存的任务列表
- 展示任务最近一次执行结果
- 展示任务是否配置定时
- 提供快捷执行入口

任务卡片支持的操作：

- 执行
- 停止
- 编辑
- 定时
- 删除

当前实现补充：

- 已支持执行、停止、编辑、删除
- `定时` 目前仅保留入口，未落地完整配置能力
- 当前“最近执行”已通过共享执行历史状态实时更新

交互约定：

- 点击任务卡片主体：进入该任务对应的编排页
- 点击 `执行`：直接执行该任务
- 点击 `停止`：停止该任务当前执行
- 点击 `编辑`：切到编排页并加载该任务
- 点击 `定时`：进入该任务的定时配置
- 点击 `删除`：二次确认后删除

#### 编排

编排页复用现有 `WorkflowEditor`，作为当前任务的编辑工作区。

主要职责：

- 编辑工作流节点和连线
- 节点配置
- 导入 / 导出工作流
- 手动执行工作流
- 示例工作流加载
- 许可证与广告等编辑页相关能力

定位说明：

- 当前阶段，编排作为一级 Tab 保留
- 其本质是“当前任务编辑区”，不是首页总容器
- 后续如果任务和定时能力进一步成熟，编排可以下沉为任务详情页

当前实现补充：

- 仍复用原有编辑器主体
- 保存入口保留在编排页，不再放在任务页
- 编辑器内部旧导航逻辑尚未清理，当前不再作为主入口，但代码仍在

#### 录制

录制页承接操作录制能力。

主要职责：

- 录制用户操作
- 生成工作流 JSON
- 成功后导入当前任务或创建新任务
- 导入成功后跳转到编排页继续调整

定位说明：

- 当前录制能力未完全稳定，因此保留一级入口，但不作为核心主流程依赖
- 页面文案可视情况标注 Beta 或实验能力

#### 更多

更多页负责收纳低频功能和工具能力。

建议包含：

- 历史记录
- 节点监控器
- 示例应用
- 设置
- 关于
- 许可证管理

设计原则：

- 更多页只作为低频能力入口
- 不承接任务主流程
- 进入具体功能后，尽量使用独立内容页，不持续堆叠大 `when` 页面容器

当前实现补充：

- 许可证管理已调整为在当前 `更多` 页面上叠加弹窗
- 视觉表现已与编排页中的许可证弹窗保持一致

## 3. 当前功能迁移映射

### 3.1 现有工作流编辑器

现状：

- `WorkflowEditor` 既是工作流编辑器，又承担首页壳子和部分导航职责

改造后：

- 保留原有编辑能力
- 从首页总入口降级为 `编排` Tab 页面
- 不再承担全局首页导航容器职责

当前代码状态：

- 启动入口已不再直接进入 `WorkflowEditor`
- 但编辑器内部原有页面切换逻辑尚未删除，属于过渡保留代码

### 3.2 侧滑栏

现状：

- 侧滑栏承担主要导航职责

改造后：

- 底部 Tab 成为主导航
- 侧滑栏如继续保留，只承接补充能力
- 后续可视实际体验决定是否弱化或移除

当前代码状态：

- 新首页结构已经不依赖侧滑作为主导航
- 侧滑相关代码仍存在，但主入口已切换

### 3.3 录制

现状：

- 已有录制面板，可生成工作流并导入

改造后：

- 单独放入 `录制` Tab
- 成功后跳到 `编排`
- 与任务管理解耦，避免录制未完善时影响主流程

### 3.4 历史 / 设置 / 监控 / 关于

现状：

- 分散在侧滑中或其他页面中

改造后：

- 收纳到 `更多`
- 通过更多页统一进入

### 3.5 任务数据存储

当前实现：

- 任务通过 `WorkflowRepository` 管理
- 当前使用 `SharedPreferences + Gson` 存储 `List<Workflow>`

说明：

- 该方案适合当前阶段的小规模任务管理
- 后续如果任务、定时、执行历史继续增长，建议迁移到 `Room`

## 4. 页面流转

### 4.1 启动流程

1. App 启动进入首页
2. 默认展示 `任务` 页
3. 用户在任务页进行管理或进入编排

### 4.2 任务到编排

1. 用户在任务页点击任务卡片或编辑按钮
2. 设置当前任务上下文
3. 切换到 `编排`
4. 编排页加载该任务对应工作流

### 4.3 录制到编排

1. 用户进入 `录制`
2. 完成录制并生成工作流
3. 将工作流导入当前任务或新任务
4. 自动跳转到 `编排`
5. 用户继续调整节点配置

### 4.4 任务定时

1. 用户在任务页点击 `定时`
2. 进入该任务的定时配置入口
3. 完成配置后返回任务页
4. 任务卡片显示已配置定时状态

说明：

- 当前阶段，定时属于任务的一个配置项
- 暂不单独抽成底部 `计划` Tab
- 后续如果出现大量计划规则、统一调度需求，再单独升级为独立模块

## 5. 状态归属

为避免后续职责混乱，本次改造明确以下状态边界：

- 首页当前 Tab：由首页容器维护
- 当前选中任务：由任务页与编排页共享
- 当前工作流内容：由工作流编辑相关 ViewModel 维护
- 任务执行状态：由任务执行层维护
- 任务定时配置：挂在任务实体下管理

核心原则：

- `任务` 管“有哪些任务、要执行哪个任务”
- `编排` 管“当前任务如何编辑”
- `录制` 管“如何生成任务”
- `更多` 管“低频功能与系统配置”

当前实现补充：

- 任务执行历史已改为共享实时数据源
- 但任务执行器本身仍是全局单实例模型
- 当前仅支持单工作流执行会话

## 6. 本次改造不做的事情

本次只调整首页导航和功能归位，不在本阶段强行引入以下改造：

- 不单独引入 `计划` 底部 Tab
- 不重做底层执行引擎
- 不立即重构所有历史页面为正式导航图
- 不要求录制能力本阶段完全成熟
- 不要求多任务并发执行在本阶段完成
- 不在本阶段立即删除 `WorkflowEditor` 内旧导航壳代码

## 7. 实施顺序建议

建议按以下顺序落地：

1. 新建首页容器，提供底部 4 个 Tab
2. 将 `任务` 设为默认首页
3. 将现有 `WorkflowEditor` 下沉到 `编排`
4. 将 `RecordingControlPanel` 放入 `录制`
5. 将历史、设置、监控、关于等能力归入 `更多`
6. 为任务页补齐快捷操作：执行、停止、编辑、定时、删除
7. 在编排页补充“当前任务”上下文显示

## 8. 后续演进预留

后续如果定时能力和任务管理复杂度继续上升，可按以下方向继续演进：

- 将 `编排` 从底部 Tab 下沉为任务详情页
- 将 `定时` 从任务属性升级为独立计划中心
- 将运行中任务和执行记录拆成独立运行中心
- 将任务、定时、执行记录数据层迁移到 `Room`
- 清理 `WorkflowEditor` 中旧的首页导航壳与侧滑主导航职责

但在当前阶段，`任务 / 编排 / 录制 / 更多` 是更适合现状、实现成本更低的方案。
