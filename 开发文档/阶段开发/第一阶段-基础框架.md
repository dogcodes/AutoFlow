- [x] 数据模型设计 (WorkflowNode, NodeType, Workflow)
- [x] 状态管理 (WorkflowViewModel核心逻辑)

## 🚀 精简开发步骤

### Step 1: 核心数据模型 (20分钟)

#### 提示词 (Token优化版)
```
请帮我创建Android工作流编辑器的核心数据模型：

1. WorkflowNode数据类 - id、type、title、坐标、输入输出端口
2. NodeType枚举 - 8种基础类型
3. Workflow数据类 - 包含nodes列表
4. 基础的输入输出端口数据类

要求：
- 只生成核心功能代码
- 不需要详细注释和错误处理
- 确保代码可直接编译

请提供WorkflowModels.kt文件。
```

### Step 2: ViewModel核心逻辑 (30分钟)

要求：

### Step 3: 增强画布缩放功能 (40分钟)

#### 提示词 (手机端优化)
```
请为WorkflowEditor添加手机端缩放和平移功能：

1. CanvasState数据类 - 管理缩放(0.3x-3x)和平移状态
2. 手势检测 - detectTransformGestures支持双指缩放和单指平移
3. ZoomControls组件 - 缩放控制按钮(+/-/重置/适应屏幕)
4. 画布变换 - graphicsLayer应用缩放和偏移变换

手机端优化：
- 防止手势冲突 (缩放vs拖拽节点)
- 智能缩放级别 (25%/50%/100%/150%/200%)
- 双击快速缩放功能
- 适应屏幕自动居中

要求：
- 使用蓝色主题控制按钮
- 平滑动画过渡效果
- 确保小屏幕可用性

请提供完整的缩放功能实现。
```

### Step 4: 基础UI组件 (40分钟)
